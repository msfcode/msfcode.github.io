I / Schéma

Ici, il y aura 1 pc Windows serveur qui s’occupera de l’authentification utilisateurs. Nous allons mettre en place un mécanisme qui permettra d’attribuer de manière dynamique une adresse IP privée à tous les équipements du réseau locale.
Il y aura un Pc linux serveur qui va héberger notre site Web (qui utilisera apache2, Nginx). Nous allons installer d’autres services comme FTP, NFS, pour les partages de fichier.
Il y aura enfin un pc linux qui s’occupera du serveur téléphonique (Astérix).

II / Planification

Voici la plannification que nous avons prévu sur les deux semaines de projet. Nous l'avons modélisé sous la forme d'un diagramme de Gant.

Tâche 1 : Installation des systèmes d’exploitation

1) Installation de différent système exploitation sur 4 pc (Windows 10, Debian, Windows server)
2) Configuration des adresses IP fixes
3) Installation de service sur linux server (apache2, Nginx)
4) Configuration des miroirs sur les postes linux
5) Installation de PuTTY

Tâche 2 : NFS et partage de fichier

1) Configurer le commutateur réseau en utilisant PuTTY.
2) Configuration d’un server NFS ou Nfs Common
3) Configuration d’un client NFS
4) Installation de services FTP

Tâche 3 : Windows server

1) Configurer le serveur DHCP pour attribuer automatiquement les adresses IP aux périphériques.
2) Configurer le serveur DNS pour résoudre les noms de domaine.
3) Authentification des utilisateurs avec Windows server 2022
4) File Share sur windows server avec SMB

Tâche 4 : Serveur Asterisk (Téléphonie) et site web

1) Configurer un serveur Asterisk.
2) Vérification des taches d’avants, résoudre des problèmes, vérifier le plan s’il est respecté ou pas ...

Tâche 5 : Sécurité

1) Mettre en place des politiques de sécurité pour protéger le réseau contre les menaces internes et externes. Nous allons plus essayer de se documente sur des attaques possibles comme du DNS spoofing et comment les résoudre.
2) Démonstrations des attaques et résolutions.
3) Possibilité de trouver autres attaques comme sur actives directory ciblant les utilisateurs.
4) Mettre en place des politiques de sécurité, telles que les pares-feux et les listes de contrôle d'accès (ACL).
5) Blocage de quelques sites internet (malveillants)

Tâche 6 : Translation NAT

1) Nous allons travailler sur le routeur qui est intermédiaire : il faudra essayer de mettre en place un NAT dynamique c’est-à-dire nous allons définir une plage d’adresse publique qui sera attribuer de manière dynamique et automatique au adresses IP pives qui essayent d’accéder à l’extérieur.
2) Tests et validation
3) Effectuer des tests de connectivité et de performance pour s'assurer que le réseau fonctionne correctement.
4) Vérifier la disponibilité des services réseau, tels que l'accès à Internet et l’accès au site depuis l’extérieur
5) Valider que toutes les fonctionnalités requises ont été mises en place avec succès.
6) Essayer de trouver de nouveaux services à mettre en place, agrandir le réseaux, ou améliorer des partis de notre réseaux.
